Accessibility Statement

Our Commitment

I am committed to ensuring that akhileshmalani.com is accessible to all people, including those with disabilities. As a blind professional who relies on assistive technology daily, I understand first-hand the importance of digital accessibility. This website has been built with accessibility as a core design principle.

Conformance Status

This website aims to conform to the Web Content Accessibility Guidelines (WCAG) 2.2 Level AA. These guidelines explain how to make web content more accessible to people with a wide range of disabilities.

Accessibility Features

  • Semantic HTML — Proper use of headings, landmarks, lists, and ARIA attributes for assistive technology compatibility.
  • Keyboard Navigation — All interactive elements are fully operable via keyboard. A skip navigation link is provided.
  • Focus Indicators — Visible focus outlines on all interactive elements for keyboard users.
  • Colour Contrast — Text and interactive elements meet WCAG 2.2 Level AA contrast ratios.
  • Responsive Design — The site adapts to different screen sizes and orientations.
  • Reduced Motion — Animations are disabled when users have enabled reduced motion preferences.
  • High Contrast Mode — The site supports Windows High Contrast Mode through forced-colors media queries.
  • Screen Reader Optimised — Content is structured and labelled for optimal screen reader navigation.

Technologies Used

  • HTML5
  • CSS3 (with CSS Custom Properties)
  • JavaScript (ES5 compatible)
  • WAI-ARIA 1.2
  • Firebase Authentication (for the Resource Library)

Assistive Technologies Tested

This website has been tested with the following assistive technologies:

  • JAWS (latest version) with Chrome and Firefox on Windows
  • NVDA (latest version) with Chrome and Firefox on Windows
  • VoiceOver with Safari on macOS and iOS
  • TalkBack with Chrome on Android
  • Windows High Contrast Mode
  • Browser zoom up to 400%

Known Limitations

  • Firebase Authentication Modal — The sign-in and registration forms rely on Firebase Authentication. While the modal itself is fully accessible with focus trapping and ARIA attributes, error messages from Firebase may not always be descriptively worded.
  • Resource Library Downloads — Some downloadable resources in the Library section are placeholder links and do not yet point to actual files.
